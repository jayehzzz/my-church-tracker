# Skills Directory

This directory contains structured guidance files that help maintain consistency across the Church Tracker project.

## Available Skills

| Skill | Description | Use When |
|-------|-------------|----------|
| [add_chart_component](./add_chart_component/SKILL.md) | SVG charts with animations, tooltips, interactivity | Creating data visualizations |
| [add_convex_feature](./add_convex_feature/SKILL.md) | Full-stack Convex workflow (schema → backend → UI) | Adding new database-backed features |
| [create_svelte_component](./create_svelte_component/SKILL.md) | Svelte 5 component templates with Motion | Building new UI components |
| [data_table_customization](./data_table_customization/SKILL.md) | DataTable columns, filtering, sorting | Displaying tabular data |
| [design_system_tokens](./design_system_tokens/SKILL.md) | Colors, typography, spacing, animations | Styling any component |
| [edit_modal_patterns](./edit_modal_patterns/SKILL.md) | View/Edit modal workflows | Implementing "Edit Anywhere" |
| [form_component](./form_component/SKILL.md) | Forms with validation and loading states | Creating input forms |
| [kpi_card_patterns](./kpi_card_patterns/SKILL.md) | Interactive metric cards with trends | Building dashboard KPIs |
| [testing_patterns](./testing_patterns/SKILL.md) | Vitest component and service testing | Writing tests |

## How Skills Work

Each skill contains a `SKILL.md` file with:
- YAML frontmatter (name, description)
- Step-by-step instructions
- Code templates and examples
- Checklists for quality assurance

## Quick Reference

**Most commonly used:**
- `design_system_tokens` - Reference this for any styling work
- `create_svelte_component` - Template for new components
- `add_convex_feature` - End-to-end feature development

**Design Philosophy alignment:**
- `edit_modal_patterns` - "Edit Anywhere" principle
- `kpi_card_patterns` - "Click = Action" principle
- `add_chart_component` - "Smooth Transitions" principle
